module Bundle : sig ... end

The bundle policy is part of the RTCConfiguration for peer connections. Bundle is a feature of the Session Description Protocol (SDP) that allows you to specify a media flow to have the same source and destination addresses and ports, along with the same udnerlyign transfer protocol. WebRTC Standards has further information.

get_transceivers t gets all of the peer connection's transceivers in the order they were added. You could stop all of the transceivers associated with a peer connection using:

val remove_track : RtpSender.t -> t -> unit

remove_track sender t will remove sender from the peer connection t, this will stop sending media from the track for the local end of the connection.

val restart_ice : t -> unit

restart_ice t will request that ICE candidate gathering be redone on both ends of the connection. You might want to use this on a state change and the state is Failed.
